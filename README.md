# Unity SKFramework

SKFramework是基于Unity的一个小型开发框架，致力于提高各类项目的开发效率，提供持续维护、升级，若使用过程中有任何疑问或发现任何bug、缺陷，可以联系作者指出。

## 环境

- Unity版本：2020.3.16
- .Net API版本：4.x

## 模块简介

- Actions - 事件链模块，基于对象池的轻量事件管理系统，用于高效管理序列或并发事件，事件包含普通事件、延时事件、定时事件、条件事件等类型。
- Audio - 音频管理模块，分为背景音乐管理器和音效管理器，提供音频的播放、暂停、恢复、停止等接口，简化程序中音效、音乐的调用逻辑。
- Config - 配置管理模块，提供统一的配置数据管理与加载机制，支持多种格式的配置文件解析，简化程序中静态配置的读取与访问。
- Debugger - 调试器模块，支持打包后运行程序时日志的查看、Hierarchy层级的查看、组件的调试，提供自定义调试界面扩展能力，用于在开发过程中便捷地查看和编辑数据。
- Events - 事件模块，提供事件的发布、订阅，为代码的解耦提供支持。
- Extension - 函数拓展模块，使用this关键字封装了部分类的拓展函数，提供链式编程支持。
- FSM - 有限状态机模块，提供轻量灵活的状态管理机制，用于处理具有明确状态切换逻辑的场景（如角色行为状态、实验步骤、敌人AI状态等），通过规范化状态定义与切换规则，实现状态逻辑的解耦与复用。
- Log - 日志模块，提供灵活的日志记录系统，支持多参数格式化日志输出，便于调试与错误跟踪。
- ObjectPool - 对象池模块，提供轻量且高性能的对象复用机制，核心解决频繁创建与销毁对象导致的内存碎片、GC频繁触发、性能开销大等问题。
- Procedure - 流程模块，提供程序流程的管理机制，用于规范程序生命周期中不太阶段的逻辑分离与切换，实现流程的有序控制。
- Resource - 资源模块，提供统一、高性能、可扩展的资源加载、卸载、缓存等管理机制，适配多资源加载模式，覆盖从开发阶段到上线阶段的全流程资源管理需求。
- UI - UI模块，提供标准化、低耦合、高性能的UI管理体系，覆盖UI视图的加载、显示、隐藏、卸载全生命周期，适配从简单弹窗到复杂多界面的UI开发需求。
- WebRequest - 网络请求模块，提供统一、易用、可扩展的网络接口请求管理机制，适配登录、数据上报、配置拉取、资源下载等全场景网络需求。
